# Автотесты REST API Яндекс.Диска

Проект автоматизированных тестов для [REST API Яндекс Диска](https://yandex.ru/dev/disk/rest/).
Тесты проверяют работу методов GET, POST, PUT и DELETE на ресурсах Диска, включая позитивные и негативные сценарии.

## Стек

- **Java 25**
- **JUnit 5** — фреймворк для запуска тестов
- **REST Assured** — библиотека для тестирования REST API
- **Allure** — генерация отчётов
- **Maven** — сборка и управление зависимостями

## Что покрыто

| Метод | Эндпоинт | Проверяемые сценарии |
|-------|----------|----------------------|
| GET | `/v1/disk` | получение метаинформации о диске |
| GET | `/v1/disk/resources` | метаинформация о папке (200), несуществующий ресурс (404), без токена (401) |
| PUT | `/v1/disk/resources` | создание папки (201), папка уже существует (409), без токена (401) |
| POST | `/v1/disk/resources/copy` | копирование папки (201), путь занят (409), нет источника (404), без токена (401) |
| DELETE | `/v1/disk/resources` | удаление папки (204), несуществующая папка (404), без токена (401) |

Всего 14 тестов. Каждый тест автономен: сам готовит необходимое состояние и убирает за собой
через `@AfterEach`.

## Требования

- JDK 17 или выше (проект собран на JDK 25)
- Maven 3.6+
- Тестовый аккаунт Яндекса и OAuth-токен (см. ниже)

## Получение OAuth-токена

Токен можно получить на [REST API Яндекс Диска](https://yandex.ru/dev/disk/rest/) —
после авторизации на странице доступна кнопка для получения отладочного OAuth-токена.

> Используйте отдельный тестовый аккаунт, а не личный.

## Запуск

Токен передаётся через переменную окружения `YANDEX_DISK_TOKEN` (в репозитории он не хранится).

1. Задайте переменную окружения:

   ```bash
   export YANDEX_DISK_TOKEN="ваш_oauth_токен"
   ```

2. Запустите тесты:

   ```bash
   mvn test
   ```

Альтернативно при запуске из IntelliJ IDEA переменную окружения можно задать
в Run Configuration (поле *Environment variables*).

## Отчёт Allure

После прогона тестов сгенерировать и открыть отчёт:

```bash
mvn allure:serve
```

Отчёт сгруппирован по функциональным областям (`@Feature`), для каждого запроса
во вложениях доступны детали HTTP-запроса и ответа.

## Структура проекта

```
src/test/java/ru/yandex/disk/tests/
├── BaseTest.java          # базовая настройка: baseURI, токен, спецификация запроса
├── DiskInfoGetTest.java   # GET — информация о диске
├── ResourceGetTest.java   # GET — метаинформация о ресурсе
├── FolderPutTest.java     # PUT — создание папки
├── FolderPostTest.java    # POST — копирование папки
└── FolderDeleteTest.java  # DELETE — удаление папки
```
